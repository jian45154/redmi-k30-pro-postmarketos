### 📘 完整概念列表（按技术层级分类）

#### 🔩 第一层：固件与 SoC 原生启动（高通底层）
| 概念 | 缩写/别名 | 在你的上下文中的角色 |
| :--- | :--- | :--- |
| **Primary Boot Loader** | PBL | 芯片内只读引导，加载 XBL/SBL。你未涉及此层（设备已亮屏）。 |
| **Secondary Boot Loader** | SBL | 高通旧版二级引导，初始化 DDR/UFS。你的 LMI（骁龙 855+）实际使用 XBL。 |
| **eXtensible Boot Loader** | XBL | 高通现代平台（SM8150 起）对 SBL 的升级版。负责基础硬件初始化。 |
| **Android Boot Loader** | ABL | 最后一阶引导，读取 GPT 并加载 `boot` 分区。是你的启动链中**第一个与 GPT 交互的环节**。 |
| **Emergency Download Mode** | EDL / 9008 模式 | 高通底层救砖模式，通过 SAHARA 协议通信。你当前未落入此层（还能进 fastboot）。 |
| **SAHARA 协议** | SAHARA | 高通在 EDL 模式下传输数据/内存转储的协议。未直接使用，但属于底层背景。 |
| **Qualcomm ROM** | ROM | 芯片内部硬编码固件，上电第一行代码。 |

#### 📦 第二层：引导加载器与镜像打包格式
| 概念 | 缩写/别名 | 在你的上下文中的角色 |
| :--- | :--- | :--- |
| **Android Boot Image** | `boot.img` | ABL 唯一能识别的容器格式（含头部、内核、DTB、cmdline）。你的内核必须打为此包。 |
| **`mkbootimg`** | - | 生成 `boot.img` 的工具。你在编译 postmarketOS 时隐式调用了它。 |
| **bootshim（Copydown）** | Shim | 关键“桩程序”，用于清理 ABL 留下的内存垃圾，并将 DTB 重定位到干净地址。你的 LMI 必须包含此组件，否则无法进入内核。 |
| **Fastboot / Fastbootd** | Fastboot | 用于刷写分区的 USB 协议。你提到“fastboot 拒绝镜像”属于此层排查项。 |
| **Boot Command Line** | `cmdline` | 内核启动参数，包含 `root=PARTUUID=...`。**你的真机 cmdline 与本地 img UUID 匹配，但真机没有目标分区，是当前根因的直接触发点**。 |

#### 🌳 第三层：内核核心与设备树（DT）
| 概念 | 缩写/别名 | 在你的上下文中的角色 |
| :--- | :--- | :--- |
| **Device Tree Source** | `.dts` | 硬件拓扑文本描述。你的 LMI 对应 `qcom/sm8150-xiaomi-lmi.dts`。 |
| **Device Tree Blob** | `.dtb` | 编译后的二进制设备树，内核直接解析。你提到“lmi DTB”需正确交付。 |
| **Flattened Device Tree** | FDT | DTB 在内存中展开的固定格式。 |
| **`compatible` 匹配** | - | 内核通过此字符串（如 `qcom,sm8150`）匹配驱动。属于背景知识。 |
| **Earlycon** | `earlycon` | 内核极早期串口输出，用于调试。**你的设备有 earlycon 输出，说明 ABL/Shim/DTB 交付通过**。 |
| **`pstore`** | `/sys/fs/pstore/` | 上次内核崩溃日志存储区。你应在写入前检查此项，确认 UFS 扫描失败的具体错误码。 |
| **主线内核 vs 下游内核** | Mainline / Vendor | 主线=社区通用；下游=厂商魔改（含闭源驱动）。你当前移植的是主线内核。 |

#### 💾 第四层：存储寻址与分区表（GPT 核心域）
| 概念 | 缩写/别名 | 在你的上下文中的角色 |
| :--- | :--- | :--- |
| **GUID Partition Table** | **GPT** | **你当前排查的核心对象**。存储设备最底层的分区目录。你的本地 img 有 GPT，真机 `userdata` 内部缺少它。 |
| **嵌套分区（Nested Partition）** | 容器化磁盘 | 将一个**完整磁盘镜像（含 GPT）**写入另一个磁盘的**分区**内。这是 postmarketOS 的默认策略，也是你后续写入操作的目标形态。 |
| **主 GPT 与 备份 GPT** | Primary / Backup GPT | GPT 在磁盘开头和结尾各存一份。若目标分区小于镜像，备份 GPT 被截断会导致内核拒绝挂载。 |
| **PARTUUID** | 分区唯一 ID | GPT 中为每个分区分配的 UUID。**你的 cmdline 中的 `root=PARTUUID=...` 正是引用嵌套分区内部的 `pmOS_root` 的 ID**。 |
| **逻辑块地址** | **LBA** | **GPT 头部固定在 LBA1**（你之前测 512/4096 模式就是在查 LBA1 的物理偏移量）。 |
| **512 字节 vs 4096 字节扇区** | 逻辑扇区大小 | 决定 GPT 头部 LBA1 在闪存上的物理偏移（0x200 或 0x1000）。你**双模式探测均无 GPT**，彻底排除了偏移量错位的可能性。 |

#### 🛠️ 第五层：块设备与回环设备挂载（探测层）
| 概念 | 缩写/别名 | 在你的上下文中的角色 |
| :--- | :--- | :--- |
| **Loop Device** | `losetup` | 将镜像文件模拟为块设备。你使用 `losetup --read-only` 挂载本地 img 进行只读验证。 |
| **`--offset` 与 `--sizelimit`** | - | 精确挂载镜像内某个分区的数据（你分别挂载了 `pmOS_boot` 和 `pmOS_root` 的偏移区间）。 |
| **`/dev/disk/by-partuuid/`** | - | 内核根据 GPT 动态生成的符号链接目录。你在诊断 Shell 中应检查此目录是否为空，以验证内核是否识别到嵌套分区。 |

#### 📂 第六层：文件系统与底层检测工具
| 概念 | 缩写/别名 | 在你的上下文中的角色 |
| :--- | :--- | :--- |
| **ext4 文件系统特性** | `FEATURE_C12` | 特指 `metadata_csum_seed` 和 `orphan_file` 特性。Ubuntu 22.04 的旧 `e2fsck 1.46.5` 不认此特性，**你改用 Alpine 新版工具复核，避免了误报损坏**。 |
| **`e2fsck -n`** | 只读检查 | 强制只读检查文件系统，不修复任何内容。你正在执行此操作验证 `pmOS_boot` 和 `pmOS_root` 的完整性。 |
| **`sgdisk`** | - | 操作 GPT 的命令行工具（来自 `gdisk` 包）。你用它在本地验证 GPT 完整性，在真机上探测嵌套分区是否存在。 |
| **`lsblk`** | - | 列出块设备及容量。你**写入前必须用此命令确认真机 `userdata` 分区容量 > 1.8GB**，以防止截断。 |
| **`blkid`** | - | 探测块设备的文件系统类型和 UUID。initramfs 扫描阶段实际调用此逻辑查找 PARTUUID。 |

#### 🌱 第七层：Initramfs 与根切换（启动早期用户态）
| 概念 | 缩写/别名 | 在你的上下文中的角色 |
| :--- | :--- | :--- |
| **Initramfs（内嵌）** | `initramfs` | 内核内嵌的临时根文件系统。你的内核通过 `CONFIG_INITRAMFS_SOURCE` 内置了一个微扫描器。 |
| **`init` 扫描脚本** | - | initramfs 中的 `init` 程序，执行 `blkid` 扫描 UFS 中的 PARTUUID。**它找不到匹配项，就会掉落诊断 Shell——这正是你的当前卡点**。 |
| **`switch_root` / `pivot_root`** | - | 丢弃 initramfs，将控制权移交给真实根文件系统的 `/sbin/init`。你卡在此步之前。 |
| **诊断 Shell / 紧急模式** | - | 当根分区挂载失败时，内核掉落的只读救援环境。你目前在该 Shell 中执行探测命令。 |

### 🧠 如何使用这份列表？

1. **自上而下地排查问题**：当遇到新故障时，先确认“哪个层次出现了异常现象”。
2. **关注标有“⭐”或“你的卡点”的概念**：这些是你当前最需要深入掌握的知识点（如 **嵌套分区**、**PARTUUID**、**LBA 扇区**）。
3. **把工具类概念（如 `losetup`、`sgdisk`、`e2fsck`）变成肌肉记忆**：这些都是救砖现场的瑞士军刀。
