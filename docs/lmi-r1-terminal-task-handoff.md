# lmi r1 完整终端任务交接

## 任务目标

在 Xiaomi Redmi K30 Pro / POCO F2 Pro（代号 `lmi`，Qualcomm SM8250）上提供一套可长期实际使用的 postmarketOS 终端系统，并将其作为 `r1` 版本同步到 GitHub 仓库，发布可供目标设备使用的镜像和配套源码资料。

这里的“完整可用终端”指手机能够稳定进入真实 Linux userspace，用户可以仅依靠触摸屏和屏幕键盘完成日常命令行工作，而不是只看到内核启动、调试 shell 或临时演示界面。

## 目标系统

- 设备：Redmi K30 Pro / POCO F2 Pro。
- 设备代号：`lmi`。
- SoC：Qualcomm SM8250。
- 系统：postmarketOS。
- 当前主线目标：downstream D110/D114 兼容组合上的 `r1` 终端版本。
- 构建与设备连接环境：WSL。
- 图形环境：Weston。
- 主要交互界面：本机触摸终端与六行屏幕键盘。

## 核心交付物

- 能在 `lmi` 上启动并进入稳定终端的 boot 镜像。
- 包含完整终端环境、Weston 客户端和六行键盘的 Android sparse userdata 镜像。
- 与镜像内容一致的设备包、内核包、终端包、键盘补丁和构建源码。
- 已有 CLI 安装器在本项目中的统一入口、文档和发布内容。
- 镜像、软件包和源码的版本标识、SHA-256 清单及机器可读 manifest。
- 面向 GitHub Release 的 boot、sparse userdata、校验文件、说明文档、源码归档、许可证资料和 SBOM。
- 不包含真实设备序列号、私钥、登录密钥、网络标识、machine-id 或原始 userdata 的公开资产。

## 本机终端要求

- 启动后能够进入真实 rootfs，并完成正常 userspace 切换。
- 终端画面可读，字体、光标、滚动和窗口尺寸适合手机竖屏。
- 触摸输入稳定，按键中心、边缘和相邻键边界都能正确命中。
- Shell 能执行常用 Linux 命令、脚本、包管理和文本编辑任务。
- 本机终端不依赖网络即可使用。
- 用户安装的软件、终端设置和键盘设置能够持久保存。
- 重启后仍能回到相同的可用终端环境。
- Ctrl、Shift、Esc、Tab、Backspace、Enter、方向键和常用符号能产生正确终端序列。
- 普通方向键模式与 application cursor mode 均能正确工作。
- Ctrl+C、Ctrl+D、Ctrl+L、Ctrl+Space、Ctrl+[ 等组合键可用。
- 长按、滑动和多点触摸不会造成不可恢复的卡键或修饰键状态。

## 六行屏幕键盘布局

| 行 | 按键 |
| --- | --- |
| 第 1 行 | `Esc` `Tab` `Ctrl` `Shift` `Backspace` |
| 第 2 行 | `1` `2` `3` `4` `5` `6` `7` `8` `9` `0` `-` `=` |
| 第 3 行 | `q` `w` `e` `r` `t` `y` `u` `i` `o` `p` `[` `]` |
| 第 4 行 | `a` `s` `d` `f` `g` `h` `j` `k` `l` `;` `'` `Enter` |
| 第 5 行 | `z` `x` `c` `v` `b` `n` `m` `,` `.` `/` `\` `Shift` |
| 第 6 行 | 八列宽 `Space`，然后 `←` `↑` `↓` `→` |

键盘需要满足以下行为：

- 两个 Shift 键共享一致的修饰状态。
- Ctrl 状态清晰可见，并由下一次适用按键消费。
- `\` 的 Shift 输出为 `|`。
- 数字、字母、标点和括号都能输出基础字符与 Shift 字符。
- 专用终端键通过终端编码器产生真实 keysym 或控制序列。
- Space 保持明显大于其他按键，方向键区域能够单手触达。
- 竖屏布局完整可用；横屏时键盘宽度和按键命中区域仍合理。

## 系统能力范围

终端版本至少需要具备：

- 稳定的显示与触摸输入。
- 可用的 USB gadget 网络或等价的维护连接。
- 可选的 SSH 访问，并支持 Ed25519 公钥登录。
- 可用的 Wi-Fi 扫描、连接、地址获取和数据传输。
- 正确的本地时间、DNS 和基础网络工具。
- 可用的存储空间、日志目录、临时目录和软件安装空间。
- 正常的充电状态显示与持续运行能力。
- 出错后能够保留足够的系统日志供定位问题。

音频、蜂窝 modem、相机、传感器和完整省电能力属于更广泛的手机硬件支持范围；它们不应阻止 `r1` 被定义为完整终端版本，但不能在未具备实际功能时写成已支持。

## CLI 安装器范围

已有 CLI 安装器需要作为本项目的一部分统一维护，承担以下产品能力：

- 识别 `lmi` 终端系统及其版本。
- 安装或更新终端相关软件包和配置。
- 显示将要安装的组件、版本和磁盘占用。
- 保留用户已有配置，并能明确区分系统文件与用户数据。
- 提供非交互模式，适合 SSH 和自动化环境。
- 对重复执行保持幂等，不重复安装相同版本或破坏现有终端。
- 输出清晰的成功、失败和已安装状态。
- 如保留本地默认密码兼容项，可使用 `147147`；远程 SSH 不应依赖公开的默认密码。

## GitHub r1 发布内容

GitHub 仓库中的 `r1` 应包含：

- 构成镜像的全部可公开源码、补丁、包定义和配置。
- boot 镜像及其 SHA-256。
- Android sparse userdata 镜像及其 SHA-256、逻辑大小和展开后身份。
- 终端、键盘、设备和内核软件包的版本清单。
- 源码与二进制软件包之间的对应关系。
- 软件许可证、NOTICE 和 SPDX SBOM。
- 目标设备、版本组合、已具备能力和已知限制说明。
- CLI 安装器及其使用文档。
- 不发布展开后的 raw userdata、大型中间文件、私有日志或设备唯一信息。

## 完成定义

满足以下结果时，任务才算完成：

- `lmi` 能稳定启动到 postmarketOS rootfs，而不是停留在 initramfs 调试环境。
- 本机触摸终端可以持续使用，六行键盘的全部基础键、Shift 字符、Ctrl 组合和方向键均正确。
- 终端配置与用户安装内容在重启后仍然存在。
- USB 维护连接、SSH 和 Wi-Fi 能支持实际命令行工作。
- CLI 安装器能够在目标系统上正确识别、安装和更新终端组件。
- boot 与 userdata 形成一致的版本组合，内核、模块、固件和 rootfs 内容相互匹配。
- GitHub 仓库包含构成 `r1` 的完整公开源码和说明。
- GitHub Release 提供可识别、可校验的目标镜像与配套资料。
- 发布说明准确区分已具备功能、已知限制和未覆盖的手机硬件能力。
